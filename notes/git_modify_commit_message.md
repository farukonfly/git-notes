# ✏️ 修改 Commit 信息 (Modify Commit Message)

在 Git 中，提交信息写错了是可以修改的。根据提交是否是最新的，以及是否已经推送到远程，处理方式有所不同。

## 1. 修改最近一次的提交 (最常用)

如果你刚刚 commit 完，发现备注写错了，或者少提了一个文件，可以使用 `--amend` 参数。

### 方法 A：直接在命令行修改 (最快)
```bash
git commit --amend -m "这里是新的修正后的提交信息"
```
*   这会直接用新的信息覆盖掉上一次的提交信息。
*   **注意**：这实际上是创建了一个**新的提交** (新的 Hash 值) 来替换旧的。

### 方法 B：打开编辑器修改
```bash
git commit --amend
```
*   这会打开默认的文本编辑器（如 Vim, Nano, VS Code），让你编辑提交信息。
*   保存并关闭编辑器后，修改即生效。

---

## 2. 修改任意旧的提交 (交互式变基)

是的，你可以修改历史中**任意一次**提交的备注，只要它还没有被合并（或者你愿意承担重写历史的风险）。这需要使用 **交互式变基 (Interactive Rebase)**。

### 步骤详解

1.  **确定要修改的范围：**
    找到你想修改的那个提交的 **父提交 (前一个提交)** 的 ID。或者简单地指明它是倒数第几次。
    ```bash
    # 方式一：修改倒数第 3 次提交以来的历史
    git rebase -i HEAD~3

    # 方式二：指定 commit hash (例如修改 abc1234 之后的历史，需要指定它的父提交)
    git rebase -i abc1234^
    ```

2.  **在编辑器中选择操作：**
    运行命令后，Git 会自动打开一个文本编辑器，列出这段时间的提交记录。
    ```text
    pick f7f3f6d change my name a bit
    pick 310154e update README formatting and add blame
    pick a5f4a0d add catfile
    ```
    把你想修改备注的那行最前面的 `pick` 改为 `reword` (或缩写 `r`)。

3.  **修改备注：**
    保存并关闭编辑器。Git 会针对每一个标记为 `reword` 的提交，再次弹出一个编辑器，让你输入新的提交信息。

4.  **完成变基：**
    修改完保存关闭，Git 会自动重写这段历史。

---

## 3. ⚠️ 极其重要的警告：关于“重写历史”

**如果你的提交已经推送到远程 (Push 过了)：**

1.  **修改本地提交后，Hash 值会变**。
2.  这意味着你的本地分支和远程分支会通过“分叉”（Diverged）。
3.  你必须使用强制推送才能更新远程：
    ```bash
    git push --force
    ```

**🚨 团队协作禁忌：**
如果其他人已经拉取（Pull）了你的代码并在其基础上工作，**绝对不要**修改已 Push 的提交历史！
这会导致其他人的历史混乱。仅在**只有你一个人使用该分支**，或者你明确知道自己在做什么时才这样做。
