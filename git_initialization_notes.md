# Git 初始化与推送到远程仓库使用笔记

这是一份关于如何从零开始初始化本地项目，并将其内容推送到 GitHub 等远程仓库的简易指南和笔记总结。

## 🌟 核心流程总结
当你新建一个本地项目并希望将其托管在远程代码库时，基本都遵循以下「5步走」流程：
**初始化本地仓库 -> 暂存文件 -> 提交到本地 -> 关联远程仓库 -> 推送远程**

---

## 📝 详细步骤说明

### 1. 初始化本地仓库
```bash
git init
```
* **作用**：将当前目录转化为可被 Git 管理的本地仓库（在根目录下生成一个隐藏的 `.git` 文件夹）。

### 2. 将所有文件添加到暂存区
```bash
git add .
```
* **作用**：把当前目录下所有新增、以及有变动的文件一次性放入暂存区（Staging Area），为后续提交做准备。其中的 `.` 代表当前目录下的所有文件。

### 3. 提交文件到本地版本库
```bash
git commit -m "Initial commit"
```
* **作用**：将暂存区里的更改正式提交到本地的历史记录中。
* **细节**：`-m` 后面跟着的是此次提交的说明信息，建议写得清晰明了（诸如："Initial commit" 表示最初的提交）。

### 4. 规范默认分支名称为主分支 (main)
```bash
git branch -M main
```
* **作用**：将当前的默认分支（旧版 Git 可能是 `master`）强制重命名为 `main`。这是由于现在 GitHub 等各大平台已将新仓库的默认分支统一为 `main`，执行这一步能保证本地分支名称与远程同步。
* **细节 (`-M` 参数的作用)**：`-M` 是 `--move --force` 的简写形式。
  * `--move` (`-m`)：表示移动（重命名）一个分支。
  * `--force` (强制)：如果目标名称（这里是 `main`）已经存在一个分支，强制覆盖原有的同名分支而不报错并拒绝重命名。这能确保无论当前分支叫什么，都能成功改名为 `main`。

### 5. 关联本地与远程仓库
```bash
# 注意将 <你的用户名> 替换为您实际的 GitHub 账户名
git remote add origin https://github.com/<你的用户名>/chieperce-cmse-notes.git
```
* **作用**：在本地 Git 里注册一个远程仓库的地址，并按惯例把该地址取别名为 `origin`。以后在使用推送、拉取时，写 `origin` 就代表了这个一长串的 URL。

### 6. 推送到远程分支并建立追踪
```bash
git push -u origin main
```
* **作用**：将本地的 `main` 分支推送到名为 `origin` 的远程仓库中。
* **细节**：`-u`（即 `--set-upstream`）不仅完成了首推，还会让本地的 `main` 与远程的 `main` 建立起“长期跟踪”关系。以后更新代码只需简单输入 `git push` 或 `git pull`，不用再每次都跟上仓库名和分支名了。
